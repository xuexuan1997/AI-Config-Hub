# AI Config Hub Foundation and Core Contracts Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the strict TypeScript monorepo, shared runtime contracts, domain schemas, ports, API envelopes, and quality gates that every AI Config Hub runtime will share.

**Architecture:** This is the first independently verifiable slice of the approved modular-monolith design. `packages/shared` owns primitives and the stable error model, `packages/core` owns domain schemas and use-case ports, and `packages/api` owns transport schemas without importing Electron; apps and infrastructure packages begin as buildable public-entry stubs so dependency direction is enforceable from day one.

**Tech Stack:** Node.js LTS, pnpm workspace, TypeScript strict, Zod, Vitest, ESLint, Prettier, dependency-cruiser, GitHub Actions.

---

## MVP plan decomposition

The approved design is too broad for one safe implementation plan. Execute and verify these plans in order; later plan documents are written immediately before their phase so they can use the code and evidence produced by earlier phases.

1. Foundation and core contracts — this document.
2. Storage, scanner, and four adapter read paths.
3. Effective configuration, diagnostics, conversion, and migration preview.
4. Deployment, backup, verification, rollback, and recovery locks.
5. Git asset repository and history workflows.
6. CLI commands, JSON envelopes, task progress, and read-only mode.
7. Electron main/preload/React UI, IPC, watcher, and external-editor refresh.
8. Packaging, three-platform CI, glibc 2.28 evidence, and release manifests.
9. Full architecture acceptance audit across all required matrices.

Each phase must leave `pnpm typecheck`, `pnpm lint`, and its scoped tests green before the next phase begins.

## Phase-one completion gate

Do not begin the storage/scanner/adapter plan until all of the following evidence exists on the same commit:

- Frozen install succeeds from a clean pnpm store.
- Strict typecheck, lint, dependency rules, unit tests, and build all pass.
- Runtime schemas exist for every stable domain record and all 13 API commands.
- No renderer-facing package can import privileged infrastructure or Node built-ins.
- Shared errors serialize only allowlisted safe fields.
- Public package entries build and resolve without deep imports or path aliases.
- The evidence document explicitly limits its claims to phase one.
